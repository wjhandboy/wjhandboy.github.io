使用Git和Github搭建个人博客

发表于 2017-01-13

前言

使用Github pages服务搭建博客的优点：

  • 轻量级的博客系统，没有麻烦的配置
  • 使用标记语言，比如Markdown
  • 无需自己搭建服务器
  • 可以随意绑定自己的域名，不仔细看的话根本看不出来你的网站是基于github的
  • 根据Github的限制，对应的每个站有300MB空间
  • 博客内容可以轻松打包、转移、发布到其它平台；

原理

Github pages支持的都是静态文件，而博客里面一般存的不只是文章，还有文章的列表、分类、标签、翻页等动态内容，假如每次写完一篇文章都要手动更新博文目录和相关链接信息，操作就太过繁琐。而通过Hexo，可以把你编写的md文件，都存放在本地，然后调用相关的命令来批量完成相关页面的生成，然后再将有改动的页面提交到github。

准备Github环境

注册过程略过

1.创建仓库

登录后点击右上角的”+”号，选择New repository，下一步，在Repository name里面填写你的用户名，并以github.io结尾。例如填写wandoux.github.io，以后就在浏览器上直接输入wandoux.github.io就可以访问自己的博客了。

2.配置SSH key

为了能正常的提交你自己写的博客，就需要拥有Github写权限，我们采用SSH key来解决本地和服务器的连接。

首先，我们需要检查您的计算机中现有的 SSH 密钥。打开你的 Git Bash 并输入：

如果提示：No such file or directory 说明你是第一次使用git。需要生成一个新的SSH密钥

然后一直按回车，最终在用户目录下会生成两个文件：id_rsa、id_rsa.pub，用记事本打开id_rsa.pub文件，复制里面的内容，登录你的Github主页，进入Settings -> SSH and GPG keys -> New SSH key，将刚复制的内容粘贴到key那里，title随便填，保存。

最后，测试SSH连接是否成功
